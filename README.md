# FilmFinder — Telegram-бот рекомендаций фильмов и сериалов

FilmFinder — учебный проект на Java. Бот интегрируется с TMDb API, показывает карточки фильмов/сериалов с постерами, хранит пользовательские предпочтения в SQLite и формирует персональные рекомендации.

## Основные функции

- поиск фильмов и сериалов по названию;
- карточки с постером, описанием, рейтингом, жанрами, датой выхода и дополнительными сведениями;
- избранное;
- список «Смотреть позже»;
- отметка «Просмотрено»;
- лайк/дизлайк;
- похожие фильмы и сериалы;
- трейлеры YouTube;
- галерея кадров через media group;
- мастер первичной настройки пользователя;
- фильтры: жанры, тип контента, рейтинг, популярность, диапазон годов, язык;
- SQLite-хранилище пользовательских данных и кэша карточек;
- пагинация результатов;
- ежедневные уведомления о новинках для пользователей, включивших уведомления.

## Запуск

1. Установить JDK 17+.
2. Создать бота через BotFather и получить Telegram token.
3. Получить TMDb API key.
4. Скопировать `config.properties.example` в `config.properties`.
5. Заполнить значения:

```properties
bot.token=YOUR_TELEGRAM_BOT_TOKEN
tmdb.api.key=YOUR_TMDB_API_KEY
tmdb.language=ru-RU
app.data.dir=data
```

6. Запустить проект:

```bash
mvn compile exec:java
```

## Команды

- `/start` — запуск и первичная настройка;
- `/help` — справка;
- `/new` — новинки и тренды;
- `/recommend` — персональные рекомендации;
- `/favorites` — избранное;
- `/watchlist` — список «Смотреть позже»;
- `/settings` — настройки пользователя, включая минимальный рейтинг и минимальную популярность;
- `/notifytest` — тестовая отправка дайджеста новинок.

Также можно просто отправить название фильма или сериала текстом.

## Хранение данных

Данные хранятся в SQLite-файле `data/filmfinder.db`. При первом запуске таблицы создаются автоматически. База содержит пользователей, настройки, пользовательские действия, избранное, watchlist, просмотренное, веса жанров, кэш карточек и отметки об отправленных уведомлениях. Для баз, созданных более ранней версией проекта, недостающие поля добавляются миграцией при запуске.

## Важно

Не публикуйте `config.properties` и файл базы данных с реальными пользовательскими данными. В репозитории должен находиться только `config.properties.example`.
